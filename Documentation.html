1 · Overview

StackTrace is a forensic “snapshot-on-pause” plugin for Python debugging in JetBrains IDEs. When execution stops—via breakpoint, exception, or a manual trigger—it builds a comprehensive Markdown report and pushes it straight to your clipboard.

  • Language Support: Python 3.x (PyCharm 2023.1 – 2025.1, IntelliJ IDEA / CLion / DataGrip with the Python plugin).
  • IDE Responsiveness: All heavy work runs on IntelliJ’s pooled executor; the EDT never blocks.
  • Data Privacy: Everything is processed locally; no network I/O is performed.

Key Captured Elements

  • Call stack with code, arguments, locals.
  • Console output (STDERR & warnings).
  • Environment metadata (OS, CPU, memory, IDE & JRE versions).
  • Python interpreter & package list.
  • Thread table (PID, TID, status, current thread).
  • Called-function source (project / external libs, depth-limited).
Tip » Configure prompts in Settings → Tools → StackTrace and feed the snapshot to ChatGPT or another LLM for instant diagnosis.

2 · Installation

  1. Open Settings | Plugins ➞ Marketplace.
  2. Search for StackTrace » Install.
  3. Restart the IDE.
  4. (Paid) Activate licence under Settings | Plugins | StackTrace.

3 · Quick Start

  1. Set a breakpoint or run code that raises an exception.
  2. When the debugger pauses, StackTrace auto-fires (unless you disabled Auto-run when debugger pauses).
  3. Paste the clipboard into Slack, Jira, GitHub, or an LLM chat.
  4. Optional: Trigger manually via Tools → Run StackTrace Now or bind it to a key in Settings | Keymap.
